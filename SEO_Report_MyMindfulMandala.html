1.4 URLs Canónicas y Hreflang

Implementación Multilingüe - MUY BUENA

  • Canonical: <link rel="canonical" href="https://www.mymindfulmandala.com/it/">
  • Hreflang: Links alternos IT/EN en header
  • Estructura URL: /en/ y /it/ claras y lógicas

Problema: Redirect JavaScript en Root

La página raíz (index.html) usa JavaScript para redirigir a las versiones de idioma. Esto no es óptimo para SEO ya que los crawlers pueden tener dificultades.

Solución: Implementar redirect 301 server-side en Netlify:

# netlify.toml [[redirects]] from = "/" to = "/en/" status = 302 conditions = {Language = ["en"]} [[redirects]] from = "/" to = "/it/" status = 302 conditions = {Language = ["it"]}

2. Análisis SEO On-Page

2.1 Schema Markup

LocalBusiness Schema - IMPLEMENTADO

El schema LocalBusiness está correctamente configurado con:

  • Nombre del negocio
  • Descripción
  • Email de contacto
  • Dirección (Italia)
  • Links a redes sociales (Facebook, Instagram)
  • Rango de precios (€€)

Product Schema - AUSENTE (CRÍTICO)

El sitio es un e-commerce pero no tiene schema Product implementado. Esto significa:

  • Los productos NO aparecerán con rich snippets en Google
  • No se mostrará precio, disponibilidad ni valoraciones
  • Pérdida significativa de CTR en resultados de búsqueda

Solución - Añadir en cada producto:

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Product", "name": "Calming Ocean Mandala", "image": "https://www.mymindfulmandala.com/foto/mandala-ocean.jpg", "description": "Hand-painted blue mandala on 30x30cm canvas", "brand": { "@type": "Brand", "name": "My Mindful Mandala" }, "offers": { "@type": "Offer", "price": "140.00", "priceCurrency": "EUR", "availability": "https://schema.org/InStock", "url": "https://www.mymindfulmandala.com/en/shop.html" } } </script>

2.2 Imágenes

Alt Text - COMPLETO

Todas las imágenes tienen atributos alt descriptivos.

Nombres de Archivo - GENÉRICOS

Los archivos de imagen usan nombres genéricos:

  • hero1.png → debería ser hand-painted-mandala-blue.jpg
  • hero2.jpg → debería ser mandala-wall-decoration.jpg
  • hero3.jpg → debería ser personalized-cake-topper.jpg

Impacto: Google Image Search no puede entender el contenido de las imágenes.

Atributos Width/Height - AUSENTES

Las imágenes no tienen dimensiones especificadas, lo que afecta Core Web Vitals (CLS).

<!-- Antes --> <img src="foto/mandala.jpg" alt="Mandala"> <!-- Después --> <img src="foto/mandala.jpg" alt="Mandala" width="600" height="600" loading="lazy">

3. Análisis SEO E-commerce

3.1 Estructura de Tienda

Categorías y Filtros - BÁSICOS

El sitio tiene categorías definidas en backend (data-category) pero no expuestas en la UI:

  • mandala-blu
  • mandala-decorativi
  • accessori

Problema: Los usuarios no pueden filtrar visualmente. Google no puede rastrear categorías.

Solución: Crear páginas de categoría dedicadas (/en/shop/blue-mandalas.html)

Breadcrumbs - AUSENTES

No hay navegación breadcrumb en ninguna página. Esto afecta:

  • UX: usuarios no saben dónde están
  • SEO: Google no puede mostrar breadcrumbs en SERPs

Solución:

<nav aria-label="breadcrumb"> <ol itemscope itemtype="https://schema.org/BreadcrumbList"> <li itemprop="itemListElement" itemscope itemtype="https://schema.org/ListItem"> <a itemprop="item" href="/en/"><span itemprop="name">Home</span></a> <meta itemprop="position" content="1" /> </li> <li itemprop="itemListElement" itemscope itemtype="https://schema.org/ListItem"> <span itemprop="name">Shop</span> <meta itemprop="position" content="2" /> </li> </ol> </nav>
